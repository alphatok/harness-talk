# 简化 LLM 推理最佳实践

> 基于 [Simplify LLM Reasoning](raw-kb/simplify-llm-reasoning.tldr.md) 的核心结论

---

## 核心理念

LLM 不是推理引擎，是**模式重混器**（next-word prediction）。  
堆砌"反思""逐步推理"等指令 → 模型重混出噪音 → 准确率下降。

**唯一有效路径：清晰度 > 复杂度。**

---

## 三条铁律

### 1. 意图明确化

- prompt 写成**合同**：目标、边界、约束、成功标准
- 用 description 写清"什么时候用"，而非"干什么"
- 易混淆的指令用 **"不用于"** 划清边界

### 2. 步骤最小化

- 只保留任务真正需要的推理步骤
- 把关每个步骤：不需要就砍掉
- 标准摘要/问答/事实检索 → 零反思步骤

### 3. 结果外部化

- 不依赖模型自我纠错（那是回音室）
- 用外部校验锚定结果：
  - 检索（RAG）获取事实
  - 工具调用执行动作
  - 人工反馈闭合回路

---

## 何时加推理步骤

| 需要加 | 不需要加 |
|--------|---------|
| 多约束规划 | 标准摘要 |
| 工具链修补 | 问答 |
| 严格数据模式 | 事实检索 |

原则：匹配真实复杂性，不是默认加。

---

## Prompt 结构规范

```
# Role / Context（一句话定位）
# Goal（明确目标）
# Constraints（硬限制）
# Input（输入数据）
# Output Format（输出格式）
# Success Criteria（什么算完成）
```

---

## 常见错误

| 错误 | 后果 |
|------|------|
| prompt 臃肿 | 中间信息丢失（Lost in the Middle） |
| 堆反思步骤 | 固化错误 → 自信胡言乱语 |
| 让模型自己纠错 | 回音室效应 → 偏离更远 |
| 假设跨模型通用 | 同一 prompt 不同模型表现天差地别 |

---

## 验证流程

1. **基线**：单次直出，不加任何推理步骤
2. **步骤门控版**：只加任务必需的最小步骤
3. **外部校验版**：用检索/工具/人工验证输出

离线跑保留数据评估 → 再上线。**回归测试 > 头脑风暴。**

---

## 一句话

> 精简 prompt + 外部锚定 = 可靠输出。  
> 少即是多。
